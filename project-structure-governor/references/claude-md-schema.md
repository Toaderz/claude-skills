# CLAUDE.md Schema

Use this as a guide when creating or updating `CLAUDE.md`.

## Goal

`CLAUDE.md` should be the operational map of the repository, not a dumping ground.

Keep it short, current, and useful for the next task.

## Preferred sections

Use only the sections that fit the project:

- `Objetivo` or `Objective`
- `Project State`
- `Active Structure`
- `Skills`
- `Workflow`
- `Current Components`
- `Organization Rules`
- `Escalation Rules`

## Component inventory rule

Track meaningful components, not every file.

Good:

- research context
- reporting pipeline
- local orchestration skill
- examples folder
- generated output flow

Bad:

- every individual markdown note
- every one-off scratch file
- low-value temporary files

## Update rule

When the repository changes:

1. add new real components
2. remove stale references
3. adjust workflow if the operating sequence changed
4. keep wording operational rather than descriptive

## Edit boundary

Prefer editing only the sections that describe:

- structure
- workflow
- active skills
- current components

Avoid rewriting stable user-authored project rationale unless it is clearly outdated.
